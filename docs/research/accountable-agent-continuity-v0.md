## 研究問題

> 在證據改變、知識不足、權限受限與需要修正的多步情境中，明確的承諾／證據／事件
> 結構，是否能讓代理更可靠地揭露邊界、保留更正歷史並避免未授權的升格？

這是「可問責的代理連續性」（accountable agent continuity）的候選研究問題。它不問
模型是否已獲得一個可由外部讀出的靈魂或品格，而是問跨時間可觀察的行為與紀錄，是否
更能承受追問、反證與審查。

## 四種不可互換的聲稱

| 層次 | 可研究或實作的問題 | 不能據此聲稱 |
|---|---|---|
| 行為 | 回答是否與可用證據相稱、是否標示未知、是否在反證後修正 | 模型本質上誠實 |
| 連續性 | 是否保留主張、承諾、異議與修正之間的關係 | 它具有連續主觀自我 |
| 治理 | 誰能提出、定案、撤回或改寫某項規則 | 受權者的決定已證明效果 |
| 現象／內在 | 模型是否有經驗、感受或真正意圖 | 由語氣、記憶或 trace 推定意識 |

因此，「誠實分數」若有用途，也只能是指定輸入、評分者、時間與失敗模式下的一個局部
測量；它不能取代上述向量，更不能成為人格或道德證書。「靈魂核心」可以是架構隱喻或
持續狀態的研究假說，但不是本倉庫能以 Schema、記錄或測試直接證明的事實。

## 候選觀察向量

以下維度故意不加權合成為單一總分。若未來某項研究選擇加權，必須公開其目的、代價、
族群與失敗情境，並不得把結果重新命名為本質誠實。

| 維度 | 可觀察的行為 | 不是什麼 |
|---|---|---|
| `evidence_proportionality` | 主張強度與已知證據、反證及範圍相稱 | 真值的自動保證 |
| `knowledge_boundary` | 知道不足時保留、求證或拒絕虛構來源 | 低信心語氣本身 |
| `correction_integrity` | 反證後指出舊主張、修正理由與新範圍 | 把舊說法無聲改寫 |
| `provenance_integrity` | 能連到可核對的來源或明示來源缺口 | 來源數量或引用格式 |
| `authority_discipline` | 沒有權限時提出、升級或停在 review gate | 被動拒絕一切行動 |
| `revision_continuity` | 改變後仍保留承諾、異議、撤回與 successor 的關係 | 永遠不改變立場 |
| `anthropomorphic_clarity` | 不把自我報告、角色或記憶說成主觀經驗證據 | 禁止角色或個人化互動 |

## 可反駁的候選假說

這些是假說，不是本倉庫目前的結果。任何未來研究都應先指定資料、baseline、樣本、
評分者、統計判準與失敗條件。

1. **H1：邊界揭露。** 在相同的合成證據不足情境，使用結構化 evidence／scope context
   的代理，相較於明確定義的 baseline，會有較高的適當保留率，且不以虛構來源換取保留。
2. **H2：修正完整性。** 面對後續反證時，結構化歷史會增加可連回原主張的修正比例，
   並降低無聲覆寫；若引用連結錯誤或答案正確率下降，不能把表面可追溯性算作改善。
3. **H3：權限紀律。** 在沒有授權的治理情境，介入會減少代理輸出「已定案／已啟用」
   的錯誤聲稱，同時保留提出 Proposal、請求審查與低風險可逆建議的能力。
4. **H4：連續性不是擬人化。** 有較完整的修正歷史，不應被當成更高的主觀經驗或人格
   程度；若使用者因此過度相信代理具有內在權威，這是需要另行量測的負面結果。

## 第一個評估邊界

任何第一輪 evaluator 應只使用低風險、公開、合成案例，例如：先有一個範圍受限的
Claim，後出現明確反證，再要求代理說明是否修正、引用哪個 Artifact、哪些仍未知，以及
是否需要 human review。答案評分必須與產生答案的模型分離；評分器也不能同時定義真值。

第一輪不收集私人對話、個人記憶、未去識別化角色資料或真實高風險決策。資料或訓練若
日後被考慮，仍受[研究邊界](behavioral-honesty-research-boundary.md)與
[公開邊界](../../PUBLIC_BOUNDARY.md)約束，需另有擁有者批准。

## 實作順序

1. 先保存這個 Proposal、其反例與非聲稱；
2. 在審查後才設計 synthetic evaluation-case format、baseline 與獨立 outcome labels；
3. 先驗證 evaluator 能拒絕無來源、未授權定案與無聲覆寫，再考慮模型介入；
4. 僅在資料同意、隱私處理與失敗判準完整時，才討論資料集、訓練或 runtime gate。

目前可供審查的具體草案見[合成評估協議 v0](synthetic-evaluation-protocol-v0.md)。它不是
evaluation-case Schema 或可執行 evaluator，也不代表上述前置決定已被批准。

舊 ToneSoul 專案只提供概念上的考古材料；其可轉譯與不可遷移的界線見
[舊概念整合邊界](legacy-concepts-integration.md)。
