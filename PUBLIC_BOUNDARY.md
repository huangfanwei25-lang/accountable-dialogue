# 公開邊界

本倉庫公開的是「如何讓 AI 協作可被問責」的概念、介面、測試與經過同意的範例；不是私人記憶、原始對話或自我改寫引擎的傾倒區。

## 可以公開

- 可重現的程式、Schema、測試與基準；
- 已去識別化、已授權的範例與資料集協議；
- 經最小化處理且具有公開權限的決策摘要、Artifact 引用與驗證範圍；
- 描述限制、失敗與尚未證明之處的文件。

## 不可公開

- 原始私人對話、個人記憶、向量資料庫、session journal 或可重識別脈絡；
- API keys、密碼、tokens、部署憑證與 `.env` 內容；
- 真實運行的私有防禦閾值、深度攻擊 payload 字典、或可直接繞過安全邊界的細節；
- 從過往私人資料自動萃取並改寫規則的具體 prompt、腳本或資料；
- 被宣稱為「內部思維」的原始內容。

## 最小充分紀錄

可稽核不等於全面揭露。公開紀錄應足以回答「改了什麼、為什麼改、憑什麼改、誰批准、哪些仍未證明」，但只保存達成此目的所需的最少內容。

若原始內容因隱私、法律或安全理由必須移除，應留下最小的處置紀錄（例如受限引用、時間、權限與替代版本），而不是強留敏感原文。

## 送出前的人工檢查

在每次公開 commit 或 push 前，維護者應：

1. 檢視 staged paths，而非盲目使用批次加入；
2. 確認沒有上列受限資料；
3. 確認文件沒有把提案、實作與驗證混成一個完成宣稱；
4. 執行與該變更相稱的測試與格式檢查；
5. 為高影響、不可逆、涉及私密或權限的變更留下系統擁有者，或具可核對明確授權者的定案紀錄；委派的範圍、期限與可逆性也必須可核對。
